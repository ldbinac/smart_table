# SmartTable 批量插入测试数据脚本

## 简介

这是一个用于向 SmartTable 平台批量插入测试数据的 Python 脚本，方便进行性能测试和压力测试。支持自动创建 Base、Table 和字段，无需手动准备测试环境。

## 功能特点

- 支持批量插入 1千、1万 甚至更多条测试数据
- 自动分批处理（每批最多 1000 条，符合 API 限制）
- **自动创建资源**：支持一键自动创建 Base → Table → 字段，无需手动准备
- **两种字段配置**：`default`（任务管理场景）和 `simple`（通用场景），满足不同测试需求
- **可配置字段映射**：通过 JSON 文件灵活指定字段与数据类型的映射关系
- **顺序号数据生成**：生成的记录包含唯一顺序号，便于排查和验证
- 显示插入进度和统计信息
- 支持批次间延迟，避免触发限流

## 前置要求

- Python 3.7+
- `requests` 库

## 安装依赖

```bash
pip install requests
```

## 参数说明

| 参数               | 说明                                            | 默认值                     | 是否必需  |
| ---------------- | --------------------------------------------- | ----------------------- | ----- |
| `--url`          | API 基础地址（后端地址，不含 `/api` 前缀）                   | <http://localhost:5000> | 否     |
| `--token`        | 认证 Token（Bearer Token）                        | -                       | **是** |
| `--count`        | 要插入的总记录数                                      | 1000                    | 否     |
| `--batch-size`   | 每批插入记录数（最大 1000）                              | 1000                    | 否     |
| `--delay`        | 批次间延迟秒数                                       | 0.5                     | 否     |
| `--table-id`     | 目标表格 ID（使用 `--auto` 时自动创建，无需指定）               | -                       | 见说明   |
| `--field-config` | 字段配置 JSON 文件路径（与 `--auto` 互斥）                 | -                       | 否     |
| `--auto`         | 自动创建 Base、Table 和字段，无需提前准备表格                  | -                       | 否     |
| `--base-name`    | 自动创建的 Base 名称（与 `--auto` 配合使用）                | 性能测试数据集                 | 否     |
| `--table-name`   | 自动创建的 Table 名称（与 `--auto` 配合使用）               | 批量测试任务表                 | 否     |
| `--field-type`   | 字段配置类型（与 `--auto` 配合使用）可选值：`default`、`simple` | default                 | 否     |

> **注意**：`--table-id` 和 `--auto` 必须指定其中之一。指定 `--table-id` 时需提供已有的表格；使用 `--auto` 时无需指定 `--table-id`，脚本会自动创建。`--auto` 和 `--field-config` 不能同时使用。

## 使用方法

### 方式一：使用已有表格（手动指定表格 ID）

```bash
cd tools/batch_test
python batch_insert_test_data.py \
  --table-id "你的表格ID" \
  --token "你的认证Token" \
  --count 10000
```

### 方式二：自动创建表格（推荐）

无需提前准备 Base 和表格，脚本自动完成全部创建：

```bash
python batch_insert_test_data.py \
  --token "你的认证Token" \
  --auto \
  --count 10000
```

### 使用自定义字段映射（配合已有表格）

```bash
python batch_insert_test_data.py \
  --url "http://localhost:5000" \
  --table-id "44e3457e-57e4-4929-a8f8-18a3c8908bb4" \
  --token "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  --count 10000 \
  --batch-size 1000 \
  --delay 0.5 \
  --field-config field_config_example.json
```

### 自动创建表格并指定字段类型

```bash
python batch_insert_test_data.py \
  --token "你的认证Token" \
  --auto \
  --base-name "我的性能测试" \
  --table-name "测试任务表" \
  --field-type simple \
  --count 5000
```

### 快速测试（少量数据）

```bash
python batch_insert_test_data.py \
  --table-id "你的表格ID" \
  --token "你的认证Token" \
  --count 100 \
  --delay 0
```

## 自动创建资源模式（--auto）

使用 `--auto` 参数时，脚本会自动执行以下步骤：

1. **创建 Base**：创建一个新的 Base（名称通过 `--base-name` 指定）
2. **创建 Table**：在 Base 中创建一个新的 Table（不创建默认字段）
3. **创建字段**：根据 `--field-type` 配置创建字段

### 字段类型说明

#### default（默认配置）— 任务管理场景

创建 6 个字段，适合任务管理类表格的测试：

| 字段名称 | 字段类型 | 映射 key      | 数据生成规则                  |
| ---- | ---- | ----------- | ----------------------- |
| 任务名称 | 单行文本 | `task_name` | 从 25 个预设任务名中随机选择，带顺序号前缀 |
| 负责人  | 单行文本 | `assignee`  | 从 5 个预设用户 ID 中随机选择      |
| 创建人  | 单行文本 | `creator`   | 从 5 个预设用户 ID 中随机选择      |
| 截止日期 | 日期   | `date`      | 当前日期 ±30 天范围内的随机日期时间    |
| 所属团队 | 多选   | `team`      | 从 5 个预设团队中随机选择 1 个      |
| 优先级  | 数字   | `priority`  | 1-5 随机整数                |

#### simple（简单配置）— 通用场景

创建 6 个字段（2 文本 + 2 数字 + 2 日期），适合通用数据测试：

| 字段名称 | 字段类型 | 映射 key   | 数据生成规则               |
| ---- | ---- | -------- | -------------------- |
| 文本1  | 单行文本 | `text`   | 随机字母数字组合，带顺序号        |
| 文本2  | 单行文本 | `text`   | 随机字母数字组合，带顺序号        |
| 数字1  | 数字   | `number` | 顺序号 + 随机小数位          |
| 数字2  | 数字   | `number` | 顺序号 + 随机小数位          |
| 日期1  | 日期   | `date`   | 当前日期 ±30 天范围内的随机日期时间 |
| 日期2  | 日期   | `date`   | 当前日期 ±30 天范围内的随机日期时间 |

## 字段配置文件（--field-config）

当使用已有表格时，可通过 JSON 文件配置字段映射关系，告诉脚本每个字段生成什么类型的数据。

### 支持的字段类型值

| 映射 key      | 说明                          | 适用字段类型 |
| ----------- | --------------------------- | ------ |
| `task_name` | 任务名称（从 25 个预设任务名中随机选择，带顺序号） | 单行文本   |
| `assignee`  | 负责人（从 5 个预设用户 ID 中随机选择）     | 单行文本   |
| `creator`   | 创建人（从 5 个预设用户 ID 中随机选择）     | 单行文本   |
| `date`      | 随机日期时间（±30 天范围内）            | 日期     |
| `team`      | 团队（从 5 个预设团队中随机选择 1 个）      | 多选     |
| `priority`  | 优先级（数字 1-5）                 | 数字     |
| `text`      | 随机文本（字母数字组合，带顺序号）           | 单行文本   |
| `number`    | 顺序号 + 随机小数位                 | 数字     |

### 配置文件示例

```json
{
  "你的字段ID1（单行文本）": "task_name",
  "你的字段ID2（单行文本）": "assignee",
  "你的字段ID3（日期）": "date",
  "你的字段ID4（多选）": "team",
  "你的字段ID5（数字）": "priority",
  "你的字段ID6（数字）": "number"
}
```

## 如何获取必要参数（手动模式）

### 1. 获取 Table ID

在浏览器中打开你的表格，URL 中可以看到表格 ID：

```
http://localhost:5000/base/xxx/table/44e3457e-57e4-4929-a8f8-18a3c8908bb4
```

### 2. 获取 Auth Token

1. 打开浏览器开发者工具（F12）
2. 切换到 Network（网络）标签
3. 在 SmartTable 中进行任意操作
4. 找到任意 API 请求，在 Headers 中找到 `Authorization`
5. 复制 Bearer 后面的 token 部分

### 3. 获取字段 ID

1. 在浏览器开发者工具中，查看表格数据的 API 响应
2. 或使用浏览器的 Elements 面板查看字段元素
3. 或者在批量插入请求中查看示例数据的字段 ID

## 注意事项

1. **API 限制**: 单次批量最多 1000 条记录，脚本会自动处理分批
2. **限流保护**: 建议设置适当的延迟时间（默认 0.5 秒），避免触发 API 限流
3. **数据格式**: 请确保字段类型与表格定义匹配
4. **认证**: Token 有过期时间，过期后需要重新获取
5. **重试机制**: 单个批次失败时会自动重试最多 3 次，连续 3 个批次失败则停止插入
6. **入库时间**: 大量数据（超 1 万条）入库时，后端处理可能需要一定时间，建议观察后端日志确认数据是否完整
7. **参数互斥**: `--auto` 和 `--field-config` 不能同时使用。`--auto` 自动创建字段时有内部映射，无需外部配置
8. **端口注意**: 脚本连接的是后端 API 服务（默认端口 5000），不是前端页面端口

## 常见问题

### Q: 如何插入 1 万条数据？

A: 只需将 `--count` 参数设置为 10000：

```bash
python batch_insert_test_data.py --auto --token "xxx" --count 10000
```

### Q: 插入速度慢怎么办？

A: 可以尝试：

- 减小 `--delay` 参数（如 0.1 或 0）
- 确保网络连接良好
- 在服务端本地运行脚本

### Q: `--auto` 和 `--field-config` 有什么区别？

A: `--auto` 是自动创建资源模式，脚本会创建全新的 Base、Table 和字段，适合从零开始的测试。`--field-config` 是用于已有表格时，指定字段 ID 与数据类型的映射关系，适合在已有表格中补充测试数据。

### Q: 如何在已有表格中插入数据并自定义字段？

A: 使用 `--field-config` 指定一个 JSON 文件，文件中配置字段 ID 到数据类型的映射：

```bash
python batch_insert_test_data.py \
  --table-id "你的表格ID" \
  --token "你的认证Token" \
  --field-config my_config.json \
  --count 5000
```

### Q: 如何自定义数据生成逻辑？

A: 可以修改 `SmartTableDataGenerator` 类中的 `_generate_*` 方法来定制数据生成规则。
