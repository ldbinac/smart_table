# 字段默认值配置模块实现计划

## 一、需求分析

实现字段默认值配置模块，为每个字段添加基于其数据类型的默认值配置功能。主要需求包括：

1. **支持多种数据类型的默认值配置**
   - 文本类型：默认文本值
   - 数字类型：默认数值
   - 日期类型：默认日期或当前时间
   - 单选类型：默认某一个选项
   - 多选类型：默认某几个选项
   - 复选框：默认选中/未选中
   - 其他类型：根据特性配置

2. **配置界面要求**
   - 直观展示所有可配置字段及其当前默认值状态
   - 允许用户修改并保存默认值设置
   - 与现有字段管理功能无缝集成

3. **数据持久化**
   - 将配置结果存储至后台数据库
   - 确保数据一致性和完整性

4. **自动应用默认值**
   - 创建新记录时自动应用已配置的默认值
   - 确保所有数据新增操作遵循预设规则

**方案：使用 config 字段存储默认值配置**

```python
config = {
    "defaultValue": <具体的默认值>,
    "defaultType": "static" | "dynamic",  # 静态值或动态值（如当前时间）
    "updatedAt": "2026-04-06T00:00:00Z"

```

**优点：**
- 不需要修改数据库 schema
- 灵活扩展其他配置
- 与现有架构兼容

### 3.4 记录创建时应用默认值

#### 3.4.1 前端 recordService 修改

**任务 3.4.1.1: 创建应用默认值的辅助函数**
```typescript
function applyDefaultValues(
  tableId: string,
  initialValues: Record<string, CellValue>
): Promise<Record<string, CellValue>> {
  // 获取所有字段
  const fields = await fieldService.getFieldsByTable(tableId);
  
  // 对每个有默认值的字段，如果 initialValues 中没有提供，则应用默认值
  const valuesWithDefaults = { ...initialValues };
  for (const field of fields) {
    if (field.defaultValue !== undefined && !(field.id in valuesWithDefaults)) {
      valuesWithDefaults[field.id] = field.defaultValue;
    }
  }
  
  return valuesWithDefaults;
}
```

**任务 3.4.1.2: 修改 createRecord 方法**
- 在创建记录前调用 `applyDefaultValues`
- 确保新记录包含所有默认值

@staticmethod
def create_record(table_id: str, values: Dict[str, Any], created_by: str = None) -> Record:
    """创建记录时应用字段默认值"""
    # 获取所有字段
    fields = FieldService.get_all_fields(table_id)
    
    # 应用默认值
    final_values = dict(values) if values else {}
    for field in fields:
        if field.id not in final_values:
            default_value = field.get_default_value()
            if default_value is not None:
                final_values[str(field.id)] = default_value
    
    # 创建记录
    record = Record(table_id=table_id, values=final_values, ...)
    ...

{

### 5.3 默认值应用时机

- **创建字段时**：可设置初始默认值
- **更新字段时**：可修改默认值配置
- **创建记录时**：自动应用当前配置的默认值
- **注意**：已存在的记录不受默认值变更影响

## 六、风险与挑战

### 6.1 技术风险

1. **数据类型兼容性**
   - 风险：字段类型变更后默认值可能不兼容
   - 解决：在类型变更时验证默认值，必要时清空

2. **选项变更影响**
   - 风险：单选/多选选项被删除后默认值无效
   - 解决：定期验证默认值有效性，提供修复建议

3. **时区处理**
   - 风险：日期默认值的时区问题
   - 解决：统一使用 UTC 存储，前端按用户时区显示

### 6.2 性能考虑

1. **批量创建记录**
   - 确保默认值应用不会显著影响性能
   - 考虑缓存字段默认值配置

2. **数据库查询优化**
   - 获取字段时预加载默认值配置
   - 避免 N+1 查询问题
