# 数据存储迁移验收清单

## 验收说明

本清单用于验证数据存储从 IndexedDB 迁移到管理端数据库的完整性和正确性。每个检查项都需要在完成后打勾确认。

---

## 一、后端 API 验收

### 1.1 Base 管理接口

- [ ] POST /api/bases 创建 Base 接口正常工作
- [ ] GET /api/bases 获取所有 Base 接口正常工作
- [ ] GET /api/bases/:id 获取单个 Base 接口正常工作
- [ ] PATCH /api/bases/:id 更新 Base 接口正常工作
- [ ] DELETE /api/bases/:id 删除 Base 接口正常工作
- [ ] POST /api/bases/:id/toggle-star 切换收藏状态接口正常工作
- [ ] 删除 Base 时级联删除关联的 Table、Field、Record、View
- [ ] 接口响应符合统一响应格式规范
- [ ] 接口有适当的权限控制

### 1.2 Table 管理接口

- [ ] POST /api/bases/:baseId/tables 创建 Table 接口正常工作
- [ ] GET /api/bases/:baseId/tables 获取 Base 下所有 Table 接口正常工作
- [ ] GET /api/tables/:id 获取单个 Table 接口正常工作
- [ ] PATCH /api/tables/:id 更新 Table 接口正常工作
- [ ] DELETE /api/tables/:id 删除 Table 接口正常工作
- [ ] Table 排序功能正常工作
- [ ] 记录数统计准确更新

### 1.3 Field 管理接口

- [ ] POST /api/tables/:tableId/fields 创建 Field 接口正常工作
- [ ] GET /api/tables/:tableId/fields 获取所有 Field 接口正常工作
- [ ] PATCH /api/fields/:id 更新 Field 接口正常工作
- [ ] DELETE /api/fields/:id 删除 Field 接口正常工作
- [ ] POST /api/fields/reorder 字段排序接口正常工作
- [ ] 支持所有 22 种字段类型的创建和配置
- [ ] 系统字段受保护，不能被删除

### 1.4 Record 管理接口

- [ ] POST /api/tables/:tableId/records 创建 Record 接口正常工作
- [ ] GET /api/tables/:tableId/records 获取所有 Record 接口正常工作（支持分页）
- [ ] GET /api/records/:id 获取单个 Record 接口正常工作
- [ ] PATCH /api/records/:id 更新 Record 接口正常工作
- [ ] DELETE /api/records/:id 删除 Record 接口正常工作
- [ ] POST /api/tables/:tableId/records/batch-create 批量创建接口正常工作
- [ ] PATCH /api/records/batch-update 批量更新接口正常工作
- [ ] DELETE /api/records/batch-delete 批量删除接口正常工作
- [ ] 字段值验证正常工作
- [ ] 必填字段检查正常工作
- [ ] 自动编号字段自动生成

### 1.5 View 管理接口

- [ ] POST /api/tables/:tableId/views 创建 View 接口正常工作
- [ ] GET /api/tables/:tableId/views 获取所有 View 接口正常工作
- [ ] GET /api/views/:id 获取单个 View 接口正常工作
- [ ] PATCH /api/views/:id 更新 View 接口正常工作
- [ ] DELETE /api/views/:id 删除 View 接口正常工作
- [ ] POST /api/views/:id/set-default 设置默认视图接口正常工作
- [ ] 支持所有 6 种视图类型的配置

### 1.6 Dashboard 管理接口

- [ ] Dashboard CRUD 接口正常工作
- [ ] Dashboard 布局保存接口正常工作
- [ ] Dashboard 刷新配置接口正常工作

### 1.7 附件管理接口

- [ ] 文件上传接口正常工作（支持大文件）
- [ ] 文件下载接口正常工作

## 三、数据一致性验收

### 3.1 CRUD 操作一致性

- [ ] 创建的数据能在后端数据库正确存储
- [ ] 读取的数据与后端数据库一致
- [ ] 更新的数据能正确同步到后端
- [ ] 删除的数据能从后端正确移除

### 3.2 关联数据一致性

- [ ] Base 删除后，关联的 Table、Field、Record、View 全部被删除
- [ ] Table 删除后，关联的 Field、Record、View 全部被删除
- [ ] Field 删除后，Record 中对应字段值被清除
- [ ] View 删除不影响其他数据

### 3.3 并发操作处理

- [ ] 多人同时编辑不同记录无冲突
- [ ] 多人同时编辑同一记录有适当的冲突处理
- [ ] 网络中断后重连能正确同步
