# 退出登录功能使用指南

## 快速开始

### 1. 启动服务

确保前端和后端服务都已启动：

```bash
# 启动后端服务
cd smarttable-backend
python app.py

# 启动前端服务（新终端）
cd smart-table
npm run dev
```

### 2. 登录系统

1. 打开浏览器访问前端地址（通常是 http://localhost:5173）
2. 访问登录页面 `/login`
3. 输入邮箱和密码
4. 点击"登录"

### 3. 使用退出登录功能

#### 单设备退出

1. 登录后，点击右上角的用户菜单（显示用户名和头像）
2. 在下拉菜单中点击"退出登录"
3. 在确认对话框中点击"确定退出"
4. 系统会自动清除登录状态并跳转到登录页

#### 多设备退出

1. 点击右上角的用户菜单
2. 点击"退出所有设备"（带红色警告图标）
3. 在确认对话框中点击"确定退出"
4. 您在所有设备的登录状态都会被清除

### 4. 测试多标签页同步

1. 打开多个浏览器标签页访问系统
2. 在标签页 A 中退出登录
3. 切换到标签页 B
4. 尝试访问受保护的页面
5. 应该会自动跳转到登录页

## 验证退出成功

### 方法 1: 检查浏览器存储

1. 打开浏览器开发者工具（F12）
2. 切换到 Application/Storage 标签
3. 查看 Local Storage 和 Session Storage
4. 确认没有 `access_token` 和 `refresh_token`

### 方法 2: 尝试访问 API

```javascript
// 在浏览器控制台执行
fetch('http://localhost:5001/api/bases', {
  headers: {
    'Authorization': 'Bearer 旧的 token'
  }
})
.then(res => console.log('状态码:', res.status))
// 应该返回 401 Unauthorized
```

### 方法 3: 浏览器后退

1. 退出登录后
2. 按浏览器后退按钮
3. 应该仍然停留在登录页或首页
4. 不会回到之前的受保护页面

## 常见问题

### Q: 退出后还能看到之前的页面？
A: 这是浏览器缓存，刷新页面后会跳转到登录页。

### Q: 退出时网络错误怎么办？
A: 即使网络错误，系统也会清除本地登录状态，确保下次可以重新登录。

### Q: 如何确认退出所有设备成功？
A: 在其他设备尝试操作，应该会收到 401 错误并跳转到登录页。

### Q: 退出后 Token 真的失效了吗？
A: 是的，后端会将 Token 加入黑名单，无法再次使用。可以通过 API 测试验证。

## 技术细节

### Token 存储位置

- **记住登录**: localStorage（长期存储，多标签页共享）
- **不记住登录**: sessionStorage（会话存储，仅当前标签页）

### 退出时清除的数据

1. localStorage 中的 token
2. sessionStorage 中的 token
3. authStore 中的用户信息
4. authStore 中的认证状态

### 多标签页同步原理

使用 CustomEvent 机制：
1. 标签页 A 退出时触发 `user-logout` 事件
2. 其他标签页监听该事件
3. 收到事件后清除本地状态

## 开发调试

### 查看退出日志

打开浏览器控制台，可以看到：
- 退出 API 调用
- 错误信息（如果有）
- Token 清除确认

### 模拟网络错误

在浏览器开发者工具中：
1. Network 标签
2. 选择 "Offline" 或慢速网络
3. 尝试退出登录
4. 验证本地状态仍被清除

### 测试 Token 失效

1. 退出登录前复制当前 token
2. 退出后使用该 token 调用 API
3. 应该返回 401 Unauthorized

## 最佳实践

1. **定期退出**: 在公共电脑上使用后务必退出
2. **多设备管理**: 如怀疑账号泄露，使用"退出所有设备"
3. **记住登录**: 个人设备可选择"记住登录"，公共设备不要选择
4. **检查登录状态**: 发现异常时及时退出并修改密码

## 故障排除

### 问题：退出后仍能看到数据

**解决方案**:
1. 清除浏览器缓存
2. 清除浏览器存储
3. 重新登录再退出

### 问题：无法退出（点击无反应）

**解决方案**:
1. 检查浏览器控制台是否有错误
2. 检查网络连接
3. 刷新页面后重试

### 问题：多标签页不同步

**解决方案**:
1. 检查是否使用同一浏览器
2. 检查是否启用浏览器隐私模式
3. 刷新其他标签页
