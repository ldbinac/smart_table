# SmartTable 技术架构设计文档

## 一、架构总览

### 1.1 架构设计理念

SmartTable 采用**前后端分离分层架构**，后端基于 Flask + SQLAlchemy，前端基于 Vue 3，支持 PostgreSQL/SQLite 双数据库，可选启用 Redis 缓存和基于 WebSocket 的实时协作功能。

```
┌─────────────────────────────────────────────────────────────────┐
│                         表现层 (Presentation)                    │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐   │
│  │ 表格视图 │ │ 看板视图 │ │ 日历视图 │ │ 甘特视图 │ │ 表单视图 │   │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘ └─────────┘   │
│  ┌─────────┐ ┌─────────┐ ┌─────────────────────────────────┐   │
│  │ 画廊视图 │ │ 仪表盘  │ │ 协作组件（在线用户/锁定/冲突）   │   │
│  └─────────┘ └─────────┘ └─────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────────┤
│                         业务层 (Business)                        │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐   │
│  │ 数据管理 │ │ 视图管理 │ │ 公式引擎 │ │ 筛选排序 │ │ 导入导出 │   │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘ └─────────┘   │
│  ┌─────────┐ ┌─────────┐ ┌─────────────────────────────────┐   │
│  │ 数据分组 │ │ 字段系统 │ │ 实时协作（在线状态/锁定/广播）   │   │
│  └─────────┘ └─────────┘ └─────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────────┤
│                         数据层 (Data)                            │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐   │
│  │ Base存储 │ │ Table存储│ │ Field存储│ │Record存储│ │View存储 │   │
```

### 2.1 数据存储层 (Storage Layer)

#### 2.1.1 后端数据库模型设计

```typescript
// db/schema.ts
import Dexie, { Table } from 'dexie';

// 多维表格
export interface Base {
  id: string;
  name: string;
  description?: string;
  icon?: string;
  color?: string;
  isStarred: boolean;
  createdAt: number;
  updatedAt: number;
}

// 数据表
export interface Table {
  id: string;
  baseId: string;
  name: string;
  description?: string;
  primaryFieldId: string;
  recordCount: number;
  order: number;
  createdAt: number;
  updatedAt: number;
}

// 字段定义
export interface Field {
  id: string;
  tableId: string;
  name: string;
  type: FieldType;  // 22种字段类型
  options?: FieldOptions;
  isPrimary: boolean;
  isSystem: boolean;
  isRequired: boolean;
  defaultValue?: any;
  description?: string;
  order: number;
  createdAt: number;
  updatedAt: number;
}

// 数据记录
export interface Record {
  id: string;
  tableId: string;
  values: Record<string, CellValue>;
  createdAt: number;
  updatedAt: number;
  createdBy?: string;
  updatedBy?: string;
}

// 视图配置
export interface View {
  id: string;
  tableId: string;
  name: string;
  type: ViewType;  // 6种视图类型
  config: ViewConfig;
  filters: FilterCondition[];
  sorts: SortConfig[];
  groupBys: string[];
  hiddenFields: string[];
  frozenFields: string[];
  rowHeight: 'short' | 'medium' | 'tall';
  isDefault: boolean;
  order: number;
  createdAt: number;
  updatedAt: number;
}

// 仪表盘
export interface Dashboard {
  id: string;
  baseId: string;
  name: string;
  description?: string;
  widgets: WidgetConfig[];
  layout: LayoutConfig;
  createdAt: number;
  updatedAt: number;
}

// 附件存储
export interface Attachment {
  id: string;
  recordId: string;
  fieldId: string;
  name: string;
  size: number;
  type: string;
  data: Blob;
  thumbnail?: Blob;
  createdAt: number;
}

// 操作历史
export interface OperationHistory {
  id: string;
  baseId: string;
  tableId?: string;
  recordId?: string;
  fieldId?: string;
  action: 'create' | 'update' | 'delete';
  entityType: 'base' | 'table' | 'field' | 'record' | 'view';
  oldValue?: any;
  newValue?: any;
  timestamp: number;
  userId?: string;
}
```

## 十四、实时协作架构

### 14.1 架构概述

实时协作功能基于 Flask-SocketIO 实现，采用可选启用设计，默认不启动。启用后支持多用户同时在线编辑同一多维表格。

### 14.2 通信架构

```
┌──────────────────────────────────────────────────────────────────┐
│                         前端 (Vue 3)                              │

│  │ (Pinia 状态)    │  │ (Composable)    │  │ (在线/锁定/冲突) │ │
```

### 14.3 启用/禁用机制

| 控制方式 | 配置 | 说明 |
|---------|------|------|
| 命令行参数 | `python run.py --enable-realtime` 或 `-r` | 显式启用 |
| 环境变量 | `ENABLE_REALTIME=true` | Docker/生产环境推荐 |
| 配置项 | `REALTIME_ENABLED = False` | config.py 默认值 |

### 14.4 功能隔离设计

实时协作模块采用条件初始化，禁用时：

- `extensions.py`：`socketio.init_app()` 不调用
- `__init__.py`：SocketIO 事件处理器不注册
- `collaboration_service.py`：通过 `broadcast_if_enabled()` 守卫，不执行广播
- 前端：动态 import `socket.io-client`，不加载协作模块
- 前端：`GET /api/realtime/status` 返回 `{ enabled: false }`，UI 自动隐藏协作组件

### 14.5 Socket.IO 事件协议

事件命名采用 `<category>:<action>` 格式：

| 类别 | 事件 | 方向 | 说明 |
|------|------|------|------|
| Room | `room:join` | Client→Server | 加入协作房间 |
| Room | `room:leave` | Client→Server | 离开协作房间 |
| Presence | `presence:view_changed` | Client→Server | 视图切换通知 |
| Presence | `presence:cell_selected` | Client→Server | 单元格选中通知 |
| Presence | `presence:user_joined` | Server→Client | 用户加入广播 |
| Presence | `presence:user_left` | Server→Client | 用户离开广播 |
| Lock | `lock:acquire` | Client→Server | 请求获取单元格锁 |
| Lock | `lock:release` | Client→Server | 释放单元格锁 |
| Lock | `lock:acquired` | Server→Client | 锁定成功通知 |
| Lock | `lock:released` | Server→Client | 解锁通知 |
| Data | `data:record_updated` | Server→Client | 记录更新推送 |
| Data | `data:record_created` | Server→Client | 记录创建推送 |
| Data | `data:record_deleted` | Server→Client | 记录删除推送 |
| Data | `data:field_updated` | Server→Client | 字段更新推送 |
| Data | `data:field_created` | Server→Client | 字段创建推送 |
| Data | `data:field_deleted` | Server→Client | 字段删除推送 |
| Data | `data:view_updated` | Server→Client | 视图更新推送 |
| Data | `data:table_updated` | Server→Client | 表格更新推送 |
| Data | `data:table_created` | Server→Client | 表格创建推送 |
| Data | `data:table_deleted` | Server→Client | 表格删除推送 |

### 14.6 冲突解决机制

采用乐观锁（Optimistic Locking）策略：

1. 客户端编辑记录时携带 `expected_updated_at` 字段
2. 服务端比对 `updated_at` 与 `expected_updated_at`
3. 不一致时返回 HTTP 409 (Conflict)，附带服务端最新数据
4. 前端展示冲突解决对话框，用户可选择覆盖或合并
