# SmartTable Flask 后端启动指南

## 环境要求

- Python 3.11+
- pip 包管理器
- (可选) PostgreSQL 数据库
- (可选) Redis 缓存
- (可选，实时协作需要) Redis 服务

## 快速启动（Windows）

### 1. 安装Python

如果系统未安装Python，请从官网下载安装：
- 下载地址：https://www.python.org/downloads/
- 建议版本：Python 3.11.x
- **重要**：安装时勾选 "Add Python to PATH"

### 2. 创建虚拟环境

```powershell
# 进入项目目录
cd d:\_dev\fs_table\smart-table-spec\smarttable-backend

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
.\venv\Scripts\activate
```

### 3. 安装依赖

```powershell
# 确保在虚拟环境中
pip install --upgrade pip

# 安装依赖
pip install -r requirements.txt
```

### 4. 配置环境变量

项目已创建 `.env` 文件，使用SQLite数据库（无需额外安装）：

```env
FLASK_APP=run.py
FLASK_ENV=development
FLASK_DEBUG=True
SECRET_KEY=smarttable-secret-key-2024
DATABASE_URL=sqlite:///smarttable.db
JWT_SECRET_KEY=smarttable-jwt-secret-key-2024
```

### 5. 初始化数据库

```powershell
# 创建数据库表
python run.py init-db
```

### 6. 启动服务

```powershell
# 方式1：使用Python直接启动（默认不启用实时协作）
python run.py

# 方式2：启用实时协作功能
python run.py --enable-realtime
# 或使用短参数
python run.py -r

# 方式3：使用Flask命令（需先设置环境变量）
$env:FLASK_APP="run.py"
flask run --host=0.0.0.0 --port=5000
```

#### 启动参数说明

| 参数 | 短参数 | 说明 |
|------|--------|------|
| `--enable-realtime` | `-r` | 启用 WebSocket 实时协作功能 |

> **注意**：默认情况下（不添加参数），实时协作功能不会启动，系统以普通模式运行。启用实时协作时，建议同时配置 Redis 作为 SocketIO 消息队列。

#### 环境变量配置

可通过环境变量控制实时协作功能：

```env
# 启用实时协作（与 --enable-realtime 参数等效）
ENABLE_REALTIME=true

# SocketIO 消息队列（推荐使用 Redis）
SOCKETIO_MESSAGE_QUEUE=redis://localhost:6379/1

# SocketIO 心跳配置
SOCKETIO_PING_TIMEOUT=60
SOCKETIO_PING_INTERVAL=25
```

## Docker 启动（推荐）

如果已安装Docker，可以使用Docker快速启动：

```powershell
# 1. 构建镜像
docker build -t smarttable-backend .

# 2. 运行容器（不启用实时协作）
docker run -d \
  -p 5000:5000 \
  -e FLASK_ENV=development \
  -e DATABASE_URL=sqlite:///app/smarttable.db \
  -v ${PWD}/uploads:/app/uploads \
  --name smarttable-backend \
  smarttable-backend

# 3. 运行容器（启用实时协作）
docker run -d \
  -p 5000:5000 \
  -e FLASK_ENV=development \
  -e DATABASE_URL=sqlite:///app/smarttable.db \
  -e ENABLE_REALTIME=true \
  -e SOCKETIO_MESSAGE_QUEUE=redis://redis:6379/1 \
  -v ${PWD}/uploads:/app/uploads \
  --name smarttable-backend \
  smarttable-backend
```

或使用 Docker Compose：

```powershell
# 启动所有服务（包含PostgreSQL和Redis）
docker-compose up -d

# 执行数据库迁移
docker-compose --profile migrate run --rm migrate
```

## 常见问题

### 问题1：'python' 不是内部或外部命令

**解决**：
1. 检查Python是否安装：`py --version`
2. 如果安装了Python但未添加到PATH，使用完整路径：`C:\Python311\python.exe`
3. 或使用 `py` 命令：`py run.py`

### 问题2：依赖安装失败

**解决**：
```powershell
# 升级pip
python -m pip install --upgrade pip

# 使用国内镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 问题3：端口被占用

**解决**：
```powershell
# 查找占用5000端口的进程
netstat -ano | findstr 5000

# 结束进程（将<PID>替换为实际进程ID）
taskkill /PID <PID> /F

# 或使用其他端口启动
python run.py --port 5001
```

### 问题4：数据库连接失败

**解决**：
- 如果使用SQLite：确保目录有写入权限
- 如果使用PostgreSQL：检查数据库服务是否启动，连接字符串是否正确

## 生产环境部署

### 使用Gunicorn

```powershell
# 安装gunicorn（已在requirements.txt中）
pip install gunicorn

# 启动生产服务器（不启用实时协作，使用 gthread worker）
gunicorn -c gunicorn.conf.py run:app

# 启动生产服务器（启用实时协作，自动使用 eventlet worker）
ENABLE_REALTIME=true gunicorn -c gunicorn.conf.py run:app
```

> **注意**：`gunicorn.conf.py` 会根据 `ENABLE_REALTIME` 环境变量自动选择 worker 类型：
> - 未启用实时协作：使用 `gthread` worker（默认）
> - 启用实时协作：使用 `eventlet` worker（支持 WebSocket）

### 使用Waitress（Windows推荐）

```powershell
# 安装waitress
pip install waitress

# 启动服务
waitress-serve --host=0.0.0.0 --port=5000 run:app
```

## 实时协作功能说明

### 功能概述

实时协作功能允许同一多维表格的多个用户同时在线编辑，系统会实时同步数据变更。

| 功能 | 说明 |
|------|------|
| 在线状态 | 显示当前正在编辑同一表格的用户头像 |
| 视图同步 | 实时同步其他用户的视图切换和滚动位置 |
| 单元格锁定 | 正在编辑的单元格自动锁定，防止多人同时编辑 |
| 冲突检测 | 基于乐观锁的冲突检测，编辑冲突时返回 409 状态码 |
| 离线队列 | 断线时操作自动缓存（最多100条），重连后自动重放 |
| 优雅降级 | 实时协作不可用时自动降级为普通模式 |

### 启用/禁用方式

| 方式 | 命令/配置 |
|------|-----------|
| 命令行参数 | `python run.py --enable-realtime` 或 `python run.py -r` |
| 环境变量 | `ENABLE_REALTIME=true` |
| Docker | 在 docker-compose.yml 中添加 `ENABLE_REALTIME=true` |

### 依赖服务

启用实时协作时，推荐配置以下服务：

- **Redis**：用于 SocketIO 消息队列和协作状态存储（房间、在线状态、锁）
- **PostgreSQL**：用于协作会话数据持久化（可选，SQLite 也可使用）

### API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/realtime/status` | GET | 查询实时协作服务是否可用 |
