## 1. 项目概述

**Smart Table** 是一款功能强大的在线智能数据表格管理系统，类似于 Airtable / 飞书多维表格，旨在让数据管理更简单、更高效。系统支持多种视图模式、丰富的字段类型、数据分组筛选排序、仪表盘可视化、表单收集、协作分享等核心能力，适用于项目管理、客户管理、任务跟踪、库存管理等多种业务场景。

### 核心特性一览

| 特性 | 说明 |
|------|------|
| 多维表管理 | 创建、编辑、删除、收藏多维表格 |
| 22 种字段类型 | 文本、数字、日期、日期时间、单选/多选、复选框、附件、关联、公式等 |
| 6 种视图模式 | 表格、看板、日历、甘特图、画廊、表单 |
| 数据分组 | 最多 3 级分组，分组内独立编号 |
| 筛选与排序 | 多条件筛选、多字段排序，配置持久化 |
| 数据导入/导出 | 支持 CSV、Excel、JSON 格式，支持 Excel 导入创建数据表 |
| 仪表盘 | 可视化数据展示，支持多种组件 |
| 协作与分享 | 成员管理、链接分享、权限控制 |
| 实时协作 | 多人在线编辑、在线状态、单元格锁定、冲突解决 |
| 表单收集 | 自定义表单、分享收集数据 |
| 模板系统 | 10 种预置模板快速创建 |
| 主题切换 | 浅色/深色/跟随系统 |
| 响应式设计 | 适配桌面端与移动端 |
| 操作历史 | 记录所有数据变更，支持历史追溯 |

### 数据模型

系统核心数据实体包括：

- **Base（多维表）**：顶层容器，包含多个数据表和仪表盘
- **Table（数据表）**：存储结构化数据的核心单元
- **Field（字段）**：定义数据列的类型与属性
- **Record（记录）**：数据行，以字段ID为键存储值
- **View（视图）**：定义数据的展示方式与筛选排序配置
- **Dashboard（仪表盘）**：可视化数据展示面板
- **Attachment（附件）**：文件附件存储
- **OperationHistory（操作历史）**：记录数据变更

## 3. 快速开始

### 环境要求

- Node.js >= 18
- npm >= 9

### 安装与运行

#### 方式一：Docker 部署（推荐）

```bash
# 克隆仓库
git clone https://github.com/ldbinac/smart_table.git
cd smart_table

# 使用 Docker Compose 启动
docker-compose up -d

# 访问系统
# http://localhost
```

#### 方式二：手动部署

```bash
# 启动后端
cd smarttable-backend
pip install -r requirements.txt
python run.py

# 启动前端
cd ../smart-table
npm install
npm run dev
```

#### 前端开发常用命令

```bash
# 构建生产版本
npm run build

# 预览构建结果
npm run preview

# 运行测试
npm run test

# 运行测试（监听模式）
npm run test:watch

# 生成测试覆盖率
npm run test:coverage
```

### 后端配置

前端默认连接 `http://localhost:5000/api` 作为后端 API 地址。可通过环境变量 `VITE_API_BASE_URL` 修改：

```bash
# .env.development 文件
VITE_API_BASE_URL=http://your-backend-server/api
```

## 4. 功能模块说明

### 4.1 用户认证

#### 注册

1. 访问系统，点击「注册」进入注册页面
2. 填写注册信息：
   - **用户名**：3-20个字符
   - **邮箱**：有效的邮箱地址（用于密码找回和邮箱验证）
   - **密码**：至少8位，必须包含大写字母、小写字母和数字
   - **验证码**：输入图片验证码
3. 点击「注册」按钮完成注册
4. 系统会发送邮箱验证邮件，点击邮件中的链接完成验证
5. 注册成功后自动跳转到登录页面

> ⚠️ **密码要求**：密码必须包含至少一个大写字母、一个小写字母和一个数字，长度不少于8位。

#### 登录

1. 访问系统，在登录页面输入邮箱和密码
2. 输入图片验证码
3. 点击「登录」按钮
4. 登录成功后跳转到首页
5. 系统使用 JWT Token 进行身份验证，支持 Token 自动刷新

> ⚠️ **安全提示**：连续5次登录失败后，账户将被锁定15分钟。

#### 忘记密码

1. 在登录页面点击「忘记密码」链接
2. 输入注册时使用的邮箱地址和验证码
3. 系统会发送密码重置邮件到您的邮箱
4. 点击邮件中的重置链接
5. 设置新密码（需满足密码强度要求）

#### 修改密码

1. 登录系统后，点击右上角用户头像
2. 选择「设置」
3. 在设置页面中找到「修改密码」
4. 输入当前密码和新密码
5. 点击「确认修改」

#### 退出登录

1. 点击右上角用户头像
2. 选择「退出登录」
3. 系统将清除本地 Token 并跳转到登录页面

### 4.2 首页与多维表管理

首页是用户进入系统后的主界面，提供多维表格的统一管理入口。

#### 首页导航

首页顶部包含四个导航标签：

| 标签 | 功能 |
|------|------|
| 首页 | 展示收藏的多维表和所有多维表卡片 |
| 全部 | 以列表形式管理所有多维表，支持分页 |
| 模板 | 浏览和使用预置模板 |
| 分享 | 查看我分享的和我被分享的多维表 |

#### 创建多维表

1. 点击首页右上角「新建」按钮
2. 在弹出对话框中选择创建方式：
   - **创建空白多维表**：从零开始创建
   - **从模板创建**：选择预置模板快速创建
3. 填写多维表信息：
   - **名称**（必填，1-50字符）
   - **描述**（选填，最多200字符）
   - **图标**：从10个预设图标中选择
   - **颜色**：从8种预设颜色中选择
4. 点击「创建」完成

#### 管理多维表

- **编辑**：点击卡片右下角菜单 → 编辑，修改名称、描述、图标和颜色
- **删除**：点击卡片右下角菜单 → 删除（删除操作不可恢复）
- **收藏/取消收藏**：点击卡片上的星标图标
- **搜索**：在顶部搜索栏输入关键词搜索多维表
- **打开**：点击卡片直接打开多维表（在新标签页中打开）

#### 分享视图

在「分享」标签页中：

- **我分享的**：查看自己创建的分享链接，支持复制链接和删除分享
- **分享给我的**：查看其他用户分享给我的多维表

### 4.3 数据表操作

进入多维表后，左侧边栏显示数据表和仪表盘列表。

#### 创建数据表

1. 点击左侧边栏底部的「+ 新建数据表」按钮
2. 在弹出的对话框中输入数据表名称（必填）和描述（选填）
3. 点击「确定」完成创建
4. 新数据表会自动创建一个默认的文本字段和表格视图

#### 管理数据表

- **选择数据表**：在左侧边栏点击数据表名称
- **重命名**：右键菜单或侧边栏菜单 → 编辑
- **复制数据表**：在数据表管理对话框中点击「复制」
- **删除**：右键菜单或侧边栏菜单 → 删除
- **收藏**：点击数据表名称旁的星标图标
- **排序**：拖拽数据表名称前的拖拽手柄调整顺序
- **批量管理**：点击侧边栏底部「管理数据表」打开管理对话框

### 4.4 字段管理

字段定义了数据表中每一列的数据类型和属性。

#### 添加字段

1. 在表格视图中，点击工具栏的「字段」按钮
2. 在字段管理对话框中点击「添加字段」
3. 配置字段属性：
   - **字段名称**（必填）
   - **字段类型**：从20+种类型中选择
   - **是否必填**
   - **默认值**
   - **描述**
   - **类型特定选项**（如数字精度、选项列表等）
4. 点击「确定」完成创建

#### 管理字段

- **编辑字段**：在字段管理对话框中点击字段旁的编辑按钮
- **删除字段**：在字段管理对话框中点击字段旁的删除按钮（删除后数据不可恢复）
- **排序字段**：拖拽字段名称前的手柄调整顺序
- **隐藏/显示字段**：切换字段的可见性（视图级别）
- **冻结列**：右键列头 → 冻结列

---

### 4.5 视图管理

视图定义了数据的展示方式，每个数据表可以拥有多个不同类型的视图。

#### 创建视图

1. 点击表格上方的视图切换器中的「+」按钮
2. 选择视图类型（表格/看板/日历/甘特图/画廊/表单）
3. 输入视图名称
4. 点击「确定」完成创建

#### 管理视图

- **切换视图**：点击视图切换器中的视图标签
- **重命名**：右键视图标签 → 重命名
- **复制视图**：右键视图标签 → 复制（复制当前视图的所有配置）
- **删除视图**：右键视图标签 → 删除
- **设为默认**：右键视图标签 → 设为默认视图

#### 视图配置持久化

每个视图独立保存以下配置：

- 筛选条件
- 排序规则
- 分组设置
- 隐藏字段
- 冻结字段
- 行高设置

### 4.6 记录管理

记录是数据表中的数据行。

#### 添加记录

- **方式一**：点击表格底部的「+」按钮，在表格行内直接编辑
- **方式二**：点击工具栏或表格底部的「添加记录」按钮，在右侧抽屉中填写
- **方式三**：在看板/日历视图中，点击对应位置的添加按钮
- **方式四**：在分组视图中，点击分组内的添加按钮（自动填充分组字段值）

#### 编辑记录

- **行内编辑**：在表格视图中直接点击单元格进行编辑
- **抽屉编辑**：点击行展开按钮，在右侧抽屉中编辑所有字段

#### 删除记录

- **单条删除**：右键记录行 → 删除，或在记录详情抽屉中点击删除
- **批量删除**：勾选多条记录 → 右键 → 删除选中记录

#### 复制记录

- 右键记录行 → 复制记录，系统会创建一条与原记录值相同的新记录

#### 查看记录历史

- 在记录详情抽屉中，可查看该记录的修改历史

---

### 4.7 筛选功能

筛选功能允许根据条件过滤显示的记录。

#### 使用筛选

1. 点击工具栏的「筛选」按钮
2. 在筛选对话框中添加筛选条件：
   - 选择筛选字段
   - 选择操作符
   - 输入筛选值
3. 可添加多个筛选条件
4. 设置条件组合方式：**AND（全部满足）** 或 **OR（任一满足）**
5. 点击「应用」生效

#### 支持的筛选操作符

| 操作符 | 说明 | 适用字段类型 |
|--------|------|------------|
| 等于 | 值完全匹配 | 所有类型 |
| 不等于 | 值不匹配 | 所有类型 |
| 包含 | 值包含指定文本 | 文本、URL、邮箱、电话 |
| 不包含 | 值不包含指定文本 | 文本、URL、邮箱、电话 |
| 开头是 | 值以指定文本开头 | 文本 |
| 结尾是 | 值以指定文本结尾 | 文本 |
| 为空 | 值为空 | 所有类型 |
| 不为空 | 值不为空 | 所有类型 |
| 大于 | 数值大于指定值 | 数字、日期 |
| 小于 | 数值小于指定值 | 数字、日期 |
| 大于等于 | 数值大于或等于指定值 | 数字、日期 |
| 小于等于 | 数值小于或等于指定值 | 数字、日期 |
| 在范围内 | 日期在指定范围内 | 日期 |
| 早于 | 日期早于指定值 | 日期 |
| 晚于 | 日期晚于指定值 | 日期 |
| 是以下之一 | 值在指定选项中 | 单选、多选 |
| 不是以下之一 | 值不在指定选项中 | 单选、多选 |

#### 清除筛选

- 在筛选对话框中点击「清除」按钮
- 筛选条件会随视图配置持久化保存

---

### 4.8 排序功能

排序功能允许按指定字段和方向对记录排序。

#### 使用排序

1. 点击工具栏的「排序」按钮
2. 在排序对话框中添加排序条件：
   - 选择排序字段
   - 选择排序方向：升序（A→Z）或降序（Z→A）
3. 可添加多个排序条件（按优先级排列）
4. 支持拖拽调整排序条件优先级
5. 点击「应用」生效

#### 清除排序

- 在排序对话框中点击「清除」按钮
- 排序配置会随视图配置持久化保存

---

### 4.9 分组功能

分组功能允许按字段值对记录进行分组展示。

#### 使用分组

1. 点击工具栏的「分组」按钮
2. 在分组对话框中选择分组字段
3. 支持最多 **3 级分组**
4. 点击「应用」生效

#### 分组视图特性

- 每个分组可独立展开/折叠
- 分组内记录独立编号
- 可在分组内直接添加记录（自动填充分组字段值）
- 表头固定（滚动时保持可见）
- 列宽可调整
- 支持斑马纹样式
- 支持行选择
- 支持右键菜单操作（排序、冻结、隐藏、编辑字段）

#### 清除分组

- 在分组对话框中点击「清除」按钮
- 分组配置会随视图配置持久化保存

---

### 4.10 数据导入与导出

#### 导入数据到现有数据表

1. 点击工具栏的「导入」按钮
2. 在导入对话框中选择文件（支持 CSV、Excel、JSON 格式）
3. 预览导入数据
4. 配置字段映射（将文件列映射到数据表字段）
5. 设置是否跳过首行（表头行）

#### Excel 导入创建数据表

系统支持通过 Excel 文件直接创建新的数据表，自动识别字段类型并导入数据：

1. 在左侧边栏点击「+ 新建数据表」按钮
2. 选择「从 Excel 导入创建」选项
3. 上传 Excel 文件（支持 .xlsx 和 .xls 格式）
4. 系统自动解析文件内容：
   - 识别字段类型（文本、数字、日期、单选、多选、复选框等）
   - 对于单选/多选字段，自动提取选项值（多选字段支持逗号分隔）
   - 为选项自动分配颜色
5. 在字段配置步骤中：
   - 修改数据表名称和描述
   - 调整字段名称和类型
   - 设置主字段（表格第一列）
   - 选择是否同时导入数据
6. 点击「创建数据表」开始处理
7. 系统显示实时进度：
   - 创建数据表结构
   - 导入数据（带进度条显示）
8. 完成后自动打开新创建的数据表

> **多选字段选项提取**：多选字段类型会自动处理逗号分隔的值。例如单元格值为"苹果, 香蕉"，系统会提取"苹果"和"香蕉"作为两个独立选项，并自动去重。

#### 导出数据

1. 点击工具栏的「导出」按钮
2. 在导出对话框中选择：
   - **导出格式**：CSV、Excel、JSON

### 4.12b 实时协作

实时协作功能允许多个用户同时在线编辑同一多维表格，系统会实时同步数据变更。

> **注意**：实时协作功能需要后端管理员启用（通过 `--enable-realtime` 启动参数或 `ENABLE_REALTIME=true` 环境变量）。如果后端未启用此功能，协作相关界面将自动隐藏，不影响其他功能使用。

#### 在线状态

当实时协作启用时，您可以看到当前正在编辑同一表格的其他用户：

1. **用户头像**：页面右上角显示当前在线的用户头像列表
2. **光标位置**：其他用户选中的单元格会显示该用户的头像标识
3. **视图位置**：其他用户切换视图时，您可以看到其当前所在的视图

#### 单元格锁定

为防止编辑冲突，系统采用单元格锁定机制：

1. 当用户开始编辑某个单元格时，该单元格自动锁定
2. 其他用户看到该单元格标记为"正在编辑"状态，显示编辑者信息
3. 编辑完成后，锁定自动释放
4. 如果用户意外断开连接，锁定会在超时后自动释放

#### 冲突解决

当两个用户同时编辑同一条记录时，系统使用乐观锁机制检测冲突：

1. **正常保存**：如果记录未被其他人修改，保存成功
2. **冲突提示**：如果记录已被其他人修改，系统弹出冲突解决对话框
3. **解决方式**：
   - **覆盖**：使用您的修改覆盖其他人的修改
   - **刷新**：放弃您的修改，加载最新数据重新编辑

#### 连接状态

页面底部显示实时协作连接状态：

| 状态 | 图标 | 说明 |
|------|------|------|
| 已连接 | 🟢 | 实时协作正常工作 |
| 重连中 | 🟡 | 网络中断，正在尝试重新连接 |
| 已断开 | 🔴 | 连接已断开，操作将缓存到离线队列 |

#### 离线操作

当网络中断时：

1. 您的编辑操作会自动缓存到离线队列（最多100条）
2. 网络恢复后，缓存的操作会自动重放
3. 如果重放时发生冲突，会弹出冲突解决对话框

#### 实时协作功能一览

| 功能 | 说明 |
|------|------|
| 在线用户显示 | 显示当前正在编辑同一表格的用户 |
| 单元格锁定 | 编辑中的单元格自动锁定 |
| 视图同步 | 实时同步其他用户的视图切换 |
| 数据变更推送 | 记录/字段/视图/表格变更实时推送 |
| 冲突检测 | 乐观锁机制检测编辑冲突 |
| 离线队列 | 断线时操作自动缓存 |
| 优雅降级 | 协作不可用时自动降级为普通模式 |

## 6. 视图类型详解

### 表格视图 (Table View)

最基础的数据展示方式，以行列形式展示所有数据。

**特性：**
- 支持行内编辑
- 列宽可拖拽调整
- 列头右键菜单（排序、冻结、隐藏、编辑字段）
- 支持分组展示
- 行选择（单选/多选）
- 虚拟滚动（大数据量优化）
- 斑马纹和网格线

### 看板视图 (Kanban View)

以卡片形式按分组字段展示数据，适用于任务管理、状态跟踪等场景。

**配置项：**
- 分组字段：选择用于分组的单选字段
- 卡片字段：选择卡片上显示的字段
- 封面字段：选择作为卡片封面的附件字段

**操作：**
- 拖拽卡片在分组间移动
- 在分组内添加新记录
- 点击卡片编辑记录

### 日历视图 (Calendar View)

以日历形式展示含日期字段的记录。

**配置项：**
- 日期字段：选择开始日期字段
- 结束日期字段：选择结束日期字段（可选，用于跨天事件）
- 默认视图：日/周/月

**操作：**
- 点击日期添加记录
- 点击事件编辑记录
- 切换日/周/月视图

### 甘特图视图 (Gantt View)

以时间线形式展示项目进度，适用于项目管理场景。

**配置项：**
- 开始日期字段
- 结束日期字段
- 持续时间字段（可选）
- 进度字段（可选）

**操作：**
- 拖拽调整任务时间范围
- 点击任务条编辑记录
- 缩放时间轴

### 画廊视图 (Gallery View)

以大卡片形式展示记录，适用于图片展示、产品目录等场景。

**配置项：**
- 图片字段：选择作为卡面图片的附件字段
- 标题字段：选择作为卡片标题的字段

**操作：**
- 点击卡片编辑记录
- 删除记录

### 表单视图 (Form View)

将数据表转换为可填写的表单，用于数据收集。

详见 [4.13 表单功能](#413-表单功能)。

   - 操作时间
   - 操作用户
   - 操作类型（创建/更新/删除）
   - 变更字段及前后值

### 操作历史内容

| 信息 | 说明 |
|------|------|
| 时间 | 操作发生的具体时间 |
| 用户 | 执行操作的用户 |
| 操作 | 创建、更新、删除 |
| 字段 | 变更的字段名称 |
| 旧值 | 变更前的值 |
| 新值 | 变更后的值 |

### 后台管理操作日志

管理员可在后台管理页面查看全系统的操作日志：

1. 访问 `/admin/logs` 页面
2. 查看所有用户的操作记录
3. 支持按时间、用户、操作类型筛选

## 9. 权限体系说明

### 用户角色

系统定义了以下用户角色（全局级别）：

| 角色 | 说明 |
|------|------|
| admin | 系统管理员，拥有所有权限 |
| workspace_admin | 工作区管理员，可管理用户 |
| editor | 编辑者，可编辑数据 |
| viewer | 查看者，仅可查看数据 |

### 多维表成员角色

在多维表级别，定义了以下成员角色：

| 角色 | 编辑数据 | 管理字段/视图 | 管理成员 | 删除多维表 |
|------|---------|-------------|---------|-----------|
| Owner | ✅ | ✅ | ✅ | ✅ |
| Admin | ✅ | ✅ | ✅ | ❌ |
| Editor | ✅ | ✅ | ❌ | ❌ |
| Commenter | ❌ | ❌ | ❌ | ❌ |
| Viewer | ❌ | ❌ | ❌ | ❌ |

### 分享权限

| 权限 | 说明 |
|------|------|
| 仅查看 (view) | 通过分享链接只能查看数据 |
| 可编辑 (edit) | 通过分享链接可以编辑数据 |

### 权限判断逻辑

1. 系统首先检查当前用户是否为多维表所有者（Owner）
2. 然后检查成员列表中的角色
3. 如果通过分享链接访问，根据分享权限确定操作范围
4. 工具栏按钮和操作入口会根据权限自动显示/隐藏

## 10. 常见问题解答（FAQ）

### Q1: 如何恢复误删的数据？

**A:** 目前删除操作不可恢复。建议在删除前开启「系统设置 → 删除前确认」选项，系统会在每次删除前弹出确认对话框。

### Q2: 数据存储在哪里？

**A:** 系统采用双重存储策略：
- **后端数据库**：主数据存储，通过 API 访问
- **IndexedDB（本地）**：离线缓存，当后端不可用时自动降级为本地数据

### Q3: 支持离线使用吗？

**A:** 部分支持。系统使用 IndexedDB 作为本地缓存，当后端 API 不可用时会尝试从本地缓存读取数据。但创建、更新等写操作需要后端支持。

### Q4: 如何修改后端 API 地址？

**A:** 修改项目根目录下的 `.env.development` 文件中的 `VITE_API_BASE_URL` 变量，然后重启开发服务器。

### Q5: 筛选和排序配置会保存吗？

**A:** 会的。所有筛选、排序、分组、隐藏字段等视图配置都会随视图持久化保存，切换视图或刷新页面后配置不会丢失。

### Q6: 分组最多支持几级？

**A:** 最多支持 3 级分组。每级分组可以独立展开/折叠。

### Q7: 公式字段支持哪些函数？

**A:** 公式字段支持常用的数学函数、字符串函数、日期函数和逻辑函数。具体支持的函数列表可在公式编辑器中查看。

### Q8: 附件上传有什么限制？

**A:** 附件字段的限制可在字段配置中自定义，包括：
- 接受的文件类型（MIME 类型）
- 单个文件最大大小
- 总大小限制
- 最大/最小文件数量
- 是否生成缩略图

### Q9: 如何批量操作记录？

**A:** 在表格视图中，勾选多条记录后，右键可进行批量删除操作。后续版本将支持更多批量操作。

### Q10: 看板视图如何移动记录？

**A:** 在看板视图中，直接拖拽卡片到目标分组列即可移动记录，系统会自动更新分组字段的值。

### Q11: 如何自定义表单的外观？

**A:** 在表单视图的配置对话框中，可以自定义表单标题、描述、提交按钮文字、可见字段和提交成功提示。

### Q12: 仪表盘数据会实时更新吗？

**A:** 仪表盘支持配置自动刷新间隔。在仪表盘设置中可开启自动刷新并设置刷新间隔。

### Q13: 忘记密码怎么办？

**A:** 在登录页面点击「忘记密码」，输入注册邮箱和验证码，系统会发送密码重置邮件到您的邮箱，点击邮件中的链接即可重置密码。

### Q14: 登录时提示账户被锁定？

**A:** 连续5次登录失败后，账户将被锁定15分钟，请稍后再试或通过「忘记密码」重置密码。

### Q15: 注册后收不到验证邮件？

**A:** 请检查垃圾邮件文件夹。如果仍未收到，可以登录后在设置中重新发送验证邮件，或联系管理员。

### Q16: 密码有什么要求？

**A:** 密码必须至少8位，包含大写字母、小写字母和数字。

### Q17: 实时协作功能如何启用？

**A:** 实时协作功能需要后端管理员启用。启用方式：
- 命令行启动时添加 `--enable-realtime` 参数：`python run.py --enable-realtime`
- 或设置环境变量：`ENABLE_REALTIME=true`
- Docker 部署时在 docker-compose.yml 中添加 `ENABLE_REALTIME=true`

如果后端未启用此功能，前端会自动隐藏协作相关界面，所有其他功能正常使用。

### Q18: 为什么看不到其他在线用户？

**A:** 可能的原因：
1. 后端未启用实时协作功能，请联系管理员启用
2. 网络连接中断，请检查页面底部的连接状态指示
3. 其他用户未打开同一表格

### Q19: 编辑冲突时如何处理？

**A:** 当系统检测到编辑冲突时，会弹出冲突解决对话框。您可以选择：
- **覆盖**：使用您的修改覆盖其他人的修改
- **刷新**：放弃您的修改，加载最新数据重新编辑

建议在多人协作时，先选中单元格查看是否有人正在编辑，避免冲突。

### Q20: Excel 导入创建数据表支持哪些字段类型？

**A:** Excel 导入创建数据表支持以下字段类型：
- 文本类型（单行文本、长文本）
- 数字类型
- 日期/日期时间类型
- 单选/多选类型（自动提取选项值）
- 复选框类型
- 邮箱、电话、URL 类型

系统会根据 Excel 列的数据内容自动识别最合适的字段类型。

### Q21: 多选字段的选项如何自动提取？

**A:** 当 Excel 列被识别为多选字段时，系统会：
1. 对每个单元格的值按英文逗号（,）分割
2. 去除每个选项的前后空格
3. 对所有选项进行去重处理
4. 最多保留 20 个唯一选项
5. 为每个选项自动分配颜色

例如，单元格值为"苹果, 香蕉, 苹果"，最终会提取出"苹果"和"香蕉"两个选项。

### Q22: Excel 导入时如何设置主字段？

**A:** 在字段配置步骤中，系统会自动建议一个主字段（通常是第一列）。您可以：
1. 点击字段行中的单选按钮设置主字段
2. 主字段将作为表格的第一列显示
3. 主字段的值在记录列表中作为记录的标识

### Q23: 导入过程中可以取消吗？

**A:** 在 Excel 导入创建数据表的过程中，系统会显示实时进度。由于这是一个原子操作，建议在导入完成前不要关闭窗口或刷新页面。如果必须中断，可以关闭浏览器标签页，但可能导致数据表创建不完整。

### Q24: 字段默认值如何配置？

**A:** 在字段编辑对话框中，可以为以下字段类型设置默认值：
- 文本字段：默认文本值
- 数字字段：默认数值
- 日期/日期时间：默认日期或"当前时间"
- 复选框：默认选中/未选中
- 单选：默认选项

配置默认值后，新建记录时该字段会自动填充默认值。

## 11. 注意事项

### 数据安全

1. **删除操作不可恢复**：删除多维表、数据表、字段或记录前请务必确认，删除后数据无法恢复
2. **定期备份**：建议定期通过导出功能备份重要数据
3. **分享链接安全**：创建分享链接时注意设置适当的权限，避免敏感数据泄露
4. **Token 安全**：系统使用 JWT Token 进行身份验证，Token 存储在本地，请勿在公共设备上保持登录

### 性能相关

1. **大数据量**：当单表记录数超过数千条时，建议使用筛选功能缩小数据范围
2. **分组性能**：3 级分组在大数据量场景下可能影响渲染性能
3. **附件大小**：大文件附件会影响加载速度，建议控制单个附件大小
4. **视图数量**：单个数据表建议不超过 10 个视图，过多视图会影响加载速度

### 浏览器兼容性

1. 推荐使用 Chrome、Edge、Firefox 等现代浏览器
2. 系统使用 IndexedDB，需要浏览器支持该特性
3. 部分功能（如文件拖拽上传）在旧版浏览器中可能不可用

### 使用建议

1. **合理规划字段**：创建数据表前先规划好字段类型和属性，避免后期频繁修改
2. **善用视图**：为不同场景创建不同视图，如一个用于日常编辑的表格视图，一个用于进度跟踪的看板视图
3. **利用模板**：新项目建议从模板创建，可以快速搭建数据结构
4. **设置默认值**：为常用字段设置默认值，提高数据录入效率
5. **使用关联字段**：跨表数据使用关联字段而非手动输入，保证数据一致性
6. **定期清理**：定期清理不需要的数据表和视图，保持工作空间整洁

### 开发相关

1. 项目使用 Vue 3 Composition API + TypeScript 开发
2. 状态管理使用 Pinia，数据流为：组件 → Store → Service → API/IndexedDB
3. 路由使用 Hash 模式（`createWebHashHistory`）
4. 数据库使用 Dexie.js 封装 IndexedDB，当前版本号为 5
5. API 请求使用 Axios，默认超时时间为 30 秒
6. 前端字段类型与后端字段类型存在映射关系（见 `normalizeFieldType` / `denormalizeFieldType`）
