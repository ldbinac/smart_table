# 打包构建指引

> 打包构建可以本地直接运行的程序，非部署于服务器上的程序。

> 若需要部署于服务器上，建议使用容器化技术（如Docker）进行部署。

# 1. 环境要求

- Python 3.11 +
- Node.js 18.x +
- Redis 7.x +

# 2. 安装依赖

```bash
pip install -r requirements.txt
```

- 安装 Node.js 依赖

```bash
npm install
```

# 3. 构建项目

```Shell
# 1. 编辑 version.json
vim version.json
# 将 "version": "1.2.3" 改为 "1.2.4"

# 2. 重新构建（所有产物自动使用新版本）
python build.py windows --skip-frontend
```

## 完整构建

```bash
cd smart-table-spec
python build.py windows  # 构建windows版本
python build.py linux  # 构建linux版本
```

## 前置检查

```bash
cd d:\_dev\fs_table\smart-table-spec
python build.py --clean-only  # 清理后重新测试
python build.py windows --skip-frontend  # 跳过前端，只测试打包
```

# 4. 验证构建

- 检查构建目录（`dist`）
- 确认所有文件都已生成

# 构建成果

```text
release/Windows/
├── SmartTable.exe      # 52.3 MB 主程序
├── start.bat           # 启动脚本
├── stop.bat            # 停止脚本
├── config/.env        # 配置模板
├── README.md          # 运行说明文档
├── data/              # SQLite 数据库目录
├── logs/              # 日志目录
└── uploads/           # 上传文件目录
```

- 包含所有可执行文件、库文件、配置文件等
- 可直接在目标平台运行
- 不依赖于任何外部环境（如Python解释器、Node.js运行环境等）
