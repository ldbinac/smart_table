## 关键技术点说明

### 1. 自定义渲染器实现方式
VTable 支持多种自定义渲染方式，建议采用以下方案：

```typescript
// 方案一：使用 customRender 配置
const columns = visibleFields.map(field => ({
  field: field.id,
  title: field.name,
  customRender: (args: any) => {
    // 返回 HTML 字符串或 DOM 元素
    return renderTableCell(args.record, field);
  }
}));

// 方案二：使用 Vue 组件插槽
// 在模板中使用 VTable 的插槽机制
```

### 2. 状态同步策略
保持现有的 Pinia store 作为单一数据源，VTable 作为纯展示层：

```
tableStore (源数据)
    ↓ (computed)
VTable 配置
    ↓
VTable 渲染
    ↓ (用户操作)
事件处理
    ↓
更新 tableStore
```

### 3. 编辑器适配方案
利用 VTable 的编辑器 API，同时复用现有的字段编辑组件：

```typescript
// 自定义编辑器
class CustomEditor {
  constructor() { /* ... */ }
  onStart() { /* 显示编辑组件 */ }
  onEnd() { /* 保存数据 */ }
}
```
