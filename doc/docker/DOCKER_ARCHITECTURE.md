# SmartTable Docker 部署架构

## 📐 架构设计

### 单镜像统一架构（推荐）

SmartTable 采用单 Docker 镜像包含前后端的统一架构设计，简化部署流程。

```
┌─────────────────────────────────────────────────────┐
│                  Docker Container                    │
│                                                      │
│  ┌──────────────┐    ┌──────────────────────────┐  │
│  │    Nginx     │───▶│      Gunicorn            │  │
│  │   (Port 80)  │    │   (Flask App Port 5000)  │  │
│  │              │    │                          │  │
│  │  - 静态文件  │    │  - REST API              │  │
│  │  - 反向代理  │    │  - 业务逻辑              │  │
│  └──────────────┘    └──────────────────────────┘  │
│                                                      │
│  ┌──────────────────────────────────────────────┐  │
│  │           Supervisor (进程管理器)             │  │
│  └──────────────────────────────────────────────┘  │
│                                                      │
│  ┌──────────────┐    ┌──────────────────────────┐  │
│  │  SQLite DB   │    │    Uploads Directory     │  │
│  │  (或外部 PG) │    │    (附件存储)             │  │
│  └──────────────┘    └──────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

## 🏗️ 镜像结构

### 多阶段构建

```dockerfile
阶段 1: frontend-builder
├── Node.js 20 Alpine
├── 安装前端依赖
├── 构建 Vue.js 应用
└── 输出：/app/frontend/dist

阶段 2: backend-builder
├── Python 3.11 Slim
├── 安装系统依赖
├── 安装 Python 包
└── 输出：/root/.local

阶段 3: production
├── Python 3.11 Slim
├── 安装 Nginx + Supervisor
├── 复制前端构建产物 → /app/static
├── 复制后端代码 → /app
├── 复制 Python 依赖
├── 配置 Nginx
├── 配置 Supervisor
└── 启动：Supervisord
```

## 🔄 请求流程

### 用户请求处理流程

```
用户请求
    │
    ▼
┌─────────────────┐
│   Nginx:80      │ ◀── 接收 HTTP 请求
└────────┬────────┘
         │
         ├─ 静态文件请求 ──▶ /app/static/ (直接返回)
         │
         ├─ /api/* ────────▶ Gunicorn:5000
         │                      │
         │                      ▼
         │                 ┌─────────────┐
         │                 │ Flask App   │
         │                 │  业务处理   │
         │                 └──────┬──────┘
         │                        │
         │                        ▼
         │                 ┌─────────────┐
         │                 │  Database   │
         │                 │   (SQLite/  │
         │                 │ PostgreSQL) │
         │                 └─────────────┘
         │
         └─ /uploads/* ───▶ /app/uploads/ (文件服务)
```

## 🎯 性能优化

### 缓存策略

```
用户请求
    │
    ▼
┌─────────────────┐
│   Nginx Cache   │ ◀── 静态资源缓存
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Redis Cache    │ ◀── API 响应缓存
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   Flask App     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Database       │
└─────────────────┘
```

### Gunicorn 配置优化

```python
# worker 数量计算
workers = (CPU 核心数 × 2) + 1

# 示例配置（不启用实时协作）
workers = 4          # 4 核 CPU
threads = 4          # 每个 worker 4 线程
worker_class = 'gthread'  # 多线程 worker
timeout = 120        # 超时时间
keepalive = 5        # 保持连接时间

# 示例配置（启用实时协作）
workers = 4          # 4 核 CPU
worker_class = 'eventlet'  # 协程 worker（支持 WebSocket）
timeout = 120        # 超时时间
keepalive = 5        # 保持连接时间
```

> **注意**：`gunicorn.conf.py` 会根据 `ENABLE_REALTIME` 环境变量自动选择 worker 类型。

## 📈 监控指标

### 关键指标

- **应用健康**: `/api/health`
- **响应时间**: Nginx 日志中的 `rt` 字段
- **错误率**: HTTP 5xx 错误占比
- **资源使用**: CPU、内存、磁盘 IO
- **数据库**: 连接数、查询性能
- **缓存**: 命中率、内存使用
