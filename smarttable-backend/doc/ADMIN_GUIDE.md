## 系统概述

管理员用户管理系统为超级管理员提供了完整的用户管理、系统配置和 operation 日志查看功能。

### 主要功能

- **用户管理**: 创建、编辑、删除用户，管理用户状态和角色
- **系统配置**: 配置系统参数、安全设置、邮件服务等
- **操作日志**: 查看所有管理操作的详细记录
- **权限控制**: 基于角色的访问控制

### 角色说明

| 角色 | 说明 | 权限 |
|------|------|------|
| 管理员 (admin) | 系统超级管理员 | 所有权限 |
| 工作区管理员 (workspace_admin) | 工作区级别管理员 | 工作区管理权限 |
| 编辑者 (editor) | 普通用户 | 数据编辑权限 |
| 查看者 (viewer) | 只读用户 | 数据查看权限 |

## 访问管理后台

### 前提条件

- 拥有管理员或工作区管理员角色
- 已登录系统

### 访问方式

1. 登录后，在左侧导航菜单中找到"系统管理"
2. 点击展开子菜单，选择：
   - **用户管理**: 管理用户账号
   - **系统配置**: 配置系统参数
   - **操作日志**: 查看操作记录

### 权限说明

- 只有管理员和工作区管理员可以看到"系统管理"菜单
- 非管理员访问管理页面会被重定向到 403 页面

## 用户管理

### 查看用户列表

1. 进入"用户管理"页面
2. 系统显示所有用户列表，包含以下信息：
   - 邮箱
   - 姓名
   - 角色
   - 状态
   - 创建时间

### 搜索和筛选

**搜索用户**
- 在搜索框输入邮箱或姓名关键词
- 点击"搜索"按钮或按回车键

**筛选用户**
- 角色筛选：选择特定角色（管理员、工作区管理员、编辑者、查看者）
- 状态筛选：选择特定状态（活跃、未激活、已暂停、已删除）

**分页**
- 每页显示 20 条记录（可调整）
- 使用分页器切换页面

### 创建用户

1. 点击"添加用户"按钮
2. 填写用户信息：
   - **邮箱**: 必填，有效的邮箱地址
   - **密码**: 必填，至少 8 位，建议包含大小写字母、数字、特殊字符
   - **姓名**: 必填
   - **角色**: 必选，从下拉列表选择
3. 点击"确定"
4. 系统创建用户并发送通知（如配置了邮件服务）

**注意事项**
- 邮箱必须唯一，不能重复
- 创建的账号默认需要首次登录修改密码
- 密码强度要求可在系统配置中调整

### 编辑用户

1. 在用户列表中找到目标用户
2. 点击"编辑"按钮
3. 修改用户信息：
   - 邮箱（不能修改）
   - 姓名
   - 角色
   - 状态
4. 点击"确定"保存

### 禁用/启用用户

**暂停用户**
1. 找到状态为"活跃"的用户
2. 点击"暂停"按钮
3. 确认操作
4. 用户状态变为"已暂停"，无法登录

**激活用户**
1. 找到状态为"已暂停"的用户
2. 点击"激活"按钮
3. 确认操作
4. 用户状态变为"活跃"，可以登录

### 重置用户密码

1. 找到目标用户
2. 点击"重置密码"按钮
3. 系统生成临时密码并显示
4. 将临时密码告知用户
5. 用户首次登录时必须修改密码

**自定义临时密码**
- 在重置密码对话框中可以输入自定义临时密码
- 临时密码也需符合密码策略要求

### 删除用户

**单个删除**
1. 找到目标用户
2. 点击"删除"按钮
3. 确认删除操作
4. 用户状态变为"已删除"（软删除）

**批量删除**
1. 勾选要删除的用户
2. 点击"批量删除"按钮
3. 确认删除操作
4. 所有选中用户状态变为"已删除"

**注意**: 删除为软删除，数据仍保留在数据库中，可以恢复。

## 系统配置

### 基础配置

**系统名称**
- 设置系统显示的名称
- 会显示在页面标题和登录页

**系统描述**
- 设置系统的描述信息
- 用于 SEO 和系统介绍

**每页记录数**
- 设置列表页面默认显示的数据条数
- 范围：10-100，步长 10

### 安全配置

**密码最小长度**
- 设置用户密码的最小字符数
- 范围：6-50
- 建议至少 8 位

**会话超时时间**
- 设置用户登录会话的有效期（分钟）
- 范围：5-1440（24 小时）
- 超时后需要重新登录

**启用双因素认证**
- 开启后用户登录需要二次验证
- 提高账户安全性

**允许注册**
- 开启后用户可以自行注册账号
- 关闭后只能由管理员创建账号

### 邮件配置

**SMTP 服务器**
- 邮件发送服务器地址
- 例如：smtp.gmail.com

**SMTP 端口**
- SMTP 服务器端口
- 通常为 587（TLS）或 465（SSL）

**发件人邮箱**
- 发送邮件时显示的邮箱地址
- 例如：noreply@example.com

**使用 SSL**
- 启用 SSL 加密连接
- 建议使用以提高安全性

### 其他配置

**启用日志记录**
- 开启后系统会记录所有操作
- 用于审计和问题排查

**日志保留天数**
- 设置操作日志的保留时间
- 范围：1-365 天
- 超过天数的日志会自动清理

**启用性能监控**
- 开启后系统会记录性能指标
- 用于系统优化和故障排查

### 保存配置

1. 修改配置项
2. 点击"保存配置"按钮
3. 系统验证配置有效性
4. 保存成功后显示提示消息

## 操作日志

### 查看日志列表

1. 进入"操作日志"页面
2. 系统显示所有操作日志，包含：
   - 操作人 ID
   - 操作类型
   - 实体类型
   - 实体 ID
   - IP 地址
   - 操作时间

### 操作类型说明

| 操作类型 | 说明 |
|----------|------|
| create | 创建资源 |
| update | 更新资源 |
| delete | 删除资源 |
| suspend | 暂停用户 |
| activate | 激活用户 |
| reset_password | 重置密码 |

### 实体类型说明

| 实体类型 | 说明 |
|----------|------|
| user | 用户 |
| config | 配置 |
| base | 多维表 |
| table | 数据表 |
| field | 字段 |
| record | 记录 |
| view | 视图 |
| dashboard | 仪表盘 |

### 筛选日志

**按操作人筛选**
- 输入操作人 ID
- 清空后显示所有操作人

**按操作类型筛选**
- 选择特定的操作类型
- 查看该类型的所有操作

**按实体类型筛选**
- 选择特定的实体类型
- 查看对该实体的所有操作

**按时间范围筛选**
- 选择开始日期和结束日期
- 查看该时间段内的操作

### 查看日志详情

1. 找到目标日志
2. 点击"详情"按钮
3. 查看详细信息：
   - 操作人 ID
   - 操作类型
   - 实体类型和 ID
   - IP 地址
   - User Agent
   - 操作时间
   - 旧值（JSON 格式）
   - 新值（JSON 格式）

### 导出日志

1. 设置筛选条件（可选）
2. 点击"导出日志"按钮
3. 系统生成 CSV 文件
4. 浏览器自动下载文件

**导出文件名**: `operation_logs_YYYYMMDD_HHMMSS.csv`

**CSV 文件包含列**:
- ID
- 操作人 ID
- 操作类型
- 实体类型
- 实体 ID
- 旧值
- 新值
- IP 地址
- User Agent
- 操作时间

## 常见问题

### Q: 为什么我看不到"系统管理"菜单？

A: 只有管理员（admin）和工作区管理员（workspace_admin）角色可以看到该菜单。如果您的角色不是这两个，请联系系统管理员。

### Q: 如何给用户分配管理员权限？

A: 
1. 进入"用户管理"页面
2. 找到目标用户
3. 点击"编辑"
4. 将角色修改为"管理员"或"工作区管理员"
5. 保存

### Q: 删除的用户可以恢复吗？

A: 可以。删除是软删除，数据仍在数据库中。联系技术人员可以通过数据库操作恢复用户状态。

### Q: 操作日志会保留多久？

A: 默认保留 30 天。可以在"系统配置" -> "其他配置"中调整"日志保留天数"。

### Q: 如何查看某个用户的所有操作？

A: 
1. 进入"操作日志"页面
2. 在筛选条件中输入用户 ID
3. 点击筛选
4. 显示该用户的所有操作记录

### Q: 重置密码后用户需要做什么？

A: 
1. 用户使用临时密码登录
2. 系统提示必须修改密码
3. 用户设置新密码
4. 之后可以使用新密码登录

### Q: 密码策略在哪里配置？

A: 在"系统配置" -> "安全配置"中可以配置：
- 密码最小长度
- 其他密码复杂度要求（如需要）

### Q: 如何批量操作用户？

A: 
1. 在用户列表中勾选要操作的用户
2. 点击相应的批量操作按钮（如"批量删除"）
3. 确认操作

### Q: 系统配置修改后立即生效吗？

A: 大部分配置修改后立即生效。部分配置（如会话超时时间）可能需要用户重新登录后生效。

### Q: 导出日志失败怎么办？

A: 
1. 检查是否有筛选条件导致数据量过大
2. 尝试缩小时间范围
3. 联系系统管理员检查服务器资源
