## 环境要求

### 系统要求

- **操作系统**: Linux (推荐 Ubuntu 20.04+), macOS, Windows (WSL2)
- **Python**: 3.11+
- **内存**: 最低 2GB RAM，推荐 4GB+
- **磁盘**: 最低 10GB 可用空间

### 依赖服务

- **PostgreSQL**: 14+ (推荐 16)
- **Redis**: 6+ (推荐 7)
- **MinIO**: 用于对象存储（可选）

> **注意**：启用实时协作功能时，Redis 为必需服务，用于 SocketIO 消息队列和协作状态存储。

## 快速开始

### 使用 Docker Compose 一键部署

```bash
# 1. 克隆项目
cd smarttable-backend

# 2. 复制环境配置文件
cp .env.example .env

# 3. 编辑环境变量
nano .env

# 4. 启动服务
docker-compose up -d

# 5. 执行数据库迁移
docker-compose --profile migrate run --rm migrate

# 6. 查看服务状态
docker-compose ps
```

服务启动后，API 可通过 http://localhost:5000 访问。

## 开发环境部署

### 1. 安装 Python 依赖

```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Linux/macOS:
source venv/bin/activate
# Windows:
venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt
```

### 2. 配置环境变量

创建 `.env` 文件：

```bash
cp .env.example .env
```

编辑 `.env` 文件：

```env
# Flask 配置
FLASK_ENV=development
FLASK_DEBUG=1
SECRET_KEY=your-secret-key-here

# 数据库配置
DATABASE_URL=postgresql://smarttable:smarttable123@localhost:5432/smarttable_dev

# Redis 配置
REDIS_URL=redis://localhost:6379/0

# JWT 配置
JWT_SECRET_KEY=your-jwt-secret-key
JWT_ACCESS_TOKEN_EXPIRES=3600
JWT_REFRESH_TOKEN_EXPIRES=604800

# 文件上传配置
UPLOAD_FOLDER=./uploads
MAX_CONTENT_LENGTH=52428800

# 实时协作配置（可选）
ENABLE_REALTIME=false
SOCKETIO_MESSAGE_QUEUE=redis://localhost:6379/1
SOCKETIO_PING_TIMEOUT=60
SOCKETIO_PING_INTERVAL=25
```

### 3. 初始化数据库

```bash
# 创建数据库（使用 psql）
createdb smarttable_dev

# 执行数据库迁移
flask db upgrade

# 或使用 Alembic 直接迁移
alembic upgrade head
```

### 4. 启动开发服务器

```bash
# 方式1: Flask 内置服务器
flask run --host=0.0.0.0 --port=5000 --reload

# 方式2: 使用 run.py
python run.py

# 方式3: 启用实时协作
python run.py --enable-realtime
```

## 生产环境部署

### 1. 使用 Docker Compose 部署

#### 准备环境文件

```bash
# 创建生产环境配置
cp .env.example .env.production

# 编辑生产环境配置
nano .env.production
```

示例生产环境配置：

```env
# 基础配置
FLASK_ENV=production
SECRET_KEY=your-very-strong-secret-key-here

# 数据库配置
DB_USER=smarttable
DB_PASSWORD=your-strong-db-password
DB_NAME=smarttable
DB_PORT=5432
DATABASE_URL=postgresql://smarttable:your-strong-db-password@postgres:5432/smarttable

# Redis 配置
REDIS_PASSWORD=your-redis-password
REDIS_URL=redis://:your-redis-password@redis:6379/0

# JWT 配置
JWT_SECRET_KEY=your-very-strong-jwt-secret

# MinIO 配置
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=your-minio-secret-key

# 其他配置
BACKEND_PORT=5000
NGINX_HTTP_PORT=80
NGINX_HTTPS_PORT=443

# 实时协作配置（可选，启用时需配置 Redis）
ENABLE_REALTIME=true
SOCKETIO_MESSAGE_QUEUE=redis://:your-redis-password@redis:6379/1
SOCKETIO_PING_TIMEOUT=60
SOCKETIO_PING_INTERVAL=25
```

#### 启动生产服务

```bash
# 启动核心服务
docker-compose up -d postgres redis minio

# 等待数据库就绪（约10秒）
sleep 10

# 执行数据库迁移
docker-compose --profile migrate run --rm migrate

# 启动后端服务
docker-compose up -d backend

# 可选: 启动 Nginx
docker-compose --profile nginx up -d nginx

# 可选: 启动 Celery
docker-compose --profile celery up -d celery-worker celery-beat
```

### 2. 手动部署（无 Docker）

#### 安装系统依赖

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install -y python3.11 python3.11-venv python3-pip postgresql-client libpq-dev

# 安装 Redis
sudo apt-get install -y redis-server

# 启动 Redis
sudo systemctl start redis
sudo systemctl enable redis
```

#### 配置 PostgreSQL

```bash
# 安装 PostgreSQL
sudo apt-get install -y postgresql-16

# 启动 PostgreSQL
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 创建数据库和用户
sudo -u postgres psql -c "CREATE USER smarttable WITH PASSWORD 'your-password';"
sudo -u postgres psql -c "CREATE DATABASE smarttable OWNER smarttable;"
sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE smarttable TO smarttable;"
```

#### 部署应用

```bash
# 创建应用目录
sudo mkdir -p /opt/smarttable
sudo chown $USER:$USER /opt/smarttable

# 克隆代码
cd /opt/smarttable
git clone <your-repo-url> .

# 创建虚拟环境
python3.11 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
pip install gunicorn

# 配置环境变量
export FLASK_ENV=production
export DATABASE_URL="postgresql://smarttable:your-password@localhost:5432/smarttable"
export SECRET_KEY="your-secret-key"
export JWT_SECRET_KEY="your-jwt-secret"

# 实时协作配置（可选）
export ENABLE_REALTIME="true"
export SOCKETIO_MESSAGE_QUEUE="redis://localhost:6379/1"

# 执行数据库迁移
flask db upgrade

# 使用 Gunicorn 启动
exec gunicorn -c gunicorn.conf.py run:app
```

#### 配置 Systemd 服务

创建服务文件 `/etc/systemd/system/smarttable.service`：

```ini
[Unit]
Description=SmartTable Flask Application
After=network.target postgresql.service redis.service

[Service]
Type=simple
User=smarttable
Group=smarttable
WorkingDirectory=/opt/smarttable
Environment="PATH=/opt/smarttable/venv/bin"
Environment="FLASK_ENV=production"
Environment="DATABASE_URL=postgresql://smarttable:password@localhost:5432/smarttable"
Environment="SECRET_KEY=your-secret-key"
Environment="JWT_SECRET_KEY=your-jwt-secret"
Environment="ENABLE_REALTIME=true"
Environment="SOCKETIO_MESSAGE_QUEUE=redis://localhost:6379/1"
ExecStart=/opt/smarttable/venv/bin/gunicorn -c gunicorn.conf.py run:app
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl start smarttable
sudo systemctl enable smarttable
```

## 数据库迁移

### 创建新迁移

```bash
# 自动检测模型变化并生成迁移脚本
flask db migrate -m "添加新字段"

# 或使用 Alembic
alembic revision --autogenerate -m "添加新字段"
```

### 执行迁移

```bash
# 升级到最新版本
flask db upgrade

# 升级到指定版本
flask db upgrade <revision>

# 降级到指定版本
flask db downgrade <revision>
```

### 查看迁移历史

```bash
# 查看当前版本
flask db current

# 查看迁移历史
flask db history
```

## 备份与恢复

### 数据库备份

```bash
# 使用 pg_dump 备份
pg_dump -h localhost -U smarttable -d smarttable -F c -f backup_$(date +%Y%m%d).dump

# 使用 Docker 备份
docker exec smarttable-postgres pg_dump -U smarttable -d smarttable -F c -f /tmp/backup.dump
docker cp smarttable-postgres:/tmp/backup.dump ./backup_$(date +%Y%m%d).dump
```

### 数据库恢复

```bash
# 使用 pg_restore 恢复
createdb smarttable_restore
pg_restore -h localhost -U smarttable -d smarttable_restore backup_20250403.dump

# 使用 Docker 恢复
docker cp backup_20250403.dump smarttable-postgres:/tmp/backup.dump
docker exec smarttable-postgres pg_restore -U smarttable -d smarttable -c /tmp/backup.dump
```

### 自动备份脚本

创建 `/opt/smarttable/scripts/backup.sh`：

```bash
#!/bin/bash

BACKUP_DIR="/opt/smarttable/backups"
DB_NAME="smarttable"
DB_USER="smarttable"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
pg_dump -h localhost -U $DB_USER -d $DB_NAME -F c -f "$BACKUP_DIR/db_backup_$DATE.dump"

# 备份上传文件
tar -czf "$BACKUP_DIR/uploads_backup_$DATE.tar.gz" /opt/smarttable/uploads

# 保留最近30天的备份
find $BACKUP_DIR -name "*.dump" -mtime +30 -delete
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete

echo "Backup completed: $DATE"
```

添加定时任务：

```bash
# 编辑 crontab
crontab -e

# 每天凌晨2点执行备份
0 2 * * * /opt/smarttable/scripts/backup.sh >> /var/log/smarttable_backup.log 2>&1
```

## 实时协作部署指南

### 概述

实时协作功能基于 Flask-SocketIO 实现，支持多用户同时在线编辑同一多维表格。该功能默认关闭，需通过启动参数或环境变量显式启用。

### 启用方式

| 方式 | 配置 |
|------|------|
| 命令行参数 | `python run.py --enable-realtime` 或 `python run.py -r` |
| 环境变量 | `ENABLE_REALTIME=true` |
| Docker | 在 docker-compose.yml 中添加 `ENABLE_REALTIME=true` |

### Gunicorn Worker 配置

`gunicorn.conf.py` 会根据 `ENABLE_REALTIME` 环境变量自动选择 worker 类型：

| 模式 | Worker 类型 | 说明 |
|------|------------|------|
| 未启用实时协作（默认） | `gthread` | 多线程 worker，适合传统 HTTP 请求 |
| 启用实时协作 | `eventlet` | 协程 worker，支持 WebSocket 长连接 |

> **重要**：启用实时协作时必须使用 `eventlet` worker，否则 WebSocket 连接无法正常工作。

### Redis 配置

启用实时协作时，强烈推荐配置 Redis：

1. **SocketIO 消息队列**：多 worker 间通过 Redis 发布/订阅同步消息
2. **协作状态存储**：在线用户、单元格锁定等状态存储在 Redis 中（带 TTL 自动过期）

```env
SOCKETIO_MESSAGE_QUEUE=redis://:password@redis-host:6379/1
```

### 数据库迁移

启用实时协作后，需执行数据库迁移以创建 `collaboration_sessions` 表：

```bash
flask db upgrade
```

### Nginx 配置

如果使用 Nginx 反向代理，需添加 WebSocket 支持：

```nginx
location /socket.io/ {
    proxy_pass http://backend:5000/socket.io/;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

### 功能隔离

实时协作功能采用模块化设计，禁用时：

- SocketIO 不会初始化，不占用额外资源
- 协作服务不会加载，无性能开销
- 前端自动检测实时协作可用性，不可用时隐藏相关 UI
- 所有其他功能（CRUD、筛选排序、导入导出等）正常工作
